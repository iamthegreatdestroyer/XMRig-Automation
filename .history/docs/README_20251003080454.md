### Command Line Scripts

Located in `scripts/` directory:

```batch

start-mining.bat

stop-mining.bat

# View logs (live tail)
view-logs.bat
```

```powershell
# Check status (detailed stats)
.\check-status.ps1

# Monitor performance (real-time dashboard)
.\monitor-performance.ps1

# Export performance data to CSV
.\monitor-performance.ps1 -ExportPath "C:\Logs\performance.csv"
```

### Auto-Start Behavior

Mining starts automatically:
- **On Windows boot** (after 30-second delay)
- **After crashes** (restarts every minute)
- **Even if no user is logged in** (runs as SYSTEM service)

To disable auto-start:
1. Open **Task Scheduler** (Win + R → `taskschd.msc`)
2. Find **"XMRig Auto Start"** task
3. Right-click → **Disable** (or Delete)

## ⚙️ Configuration

### Mining Configuration

Main config file: `C:\XMRig\config.json`

Key settings:
- **Pool**: xmrpool.eu:3333
- **Algorithm**: RandomX (rx/0)
- **Threads**: 75% (12 of 16 threads)
- **Huge Pages**: Enabled
- **Wallet**: Your XMR address

To modify settings:
1. Stop mining: `stop-mining.bat`
2. Edit `C:\XMRig\config.json`
3. Start mining: `start-mining.bat`

See [`config/CONFIG-EXPLAINED.md`](config/CONFIG-EXPLAINED.md) for detailed configuration guide.

### Adjust CPU Usage

To use **more CPU** (higher hashrate, less responsive):
```json
"max-threads-hint": 85  // Use 85% instead of 75%
```

To use **less CPU** (lower hashrate, more responsive):
```json
"max-threads-hint": 60  // Use 60% instead of 75%
```

### Change Pool

Edit `config.json`:
```json
"url": "your-pool.com:port",
"user": "your-wallet-address",
"rig-id": "your-rig-name"
```

Popular pools:
- xmrpool.eu:3333 (current)
- supportxmr.com:3333
- minexmr.com:4444
- hashvault.pro:3333

## 🔧 Troubleshooting

### Common Issues

**Mining not starting?**
- Check Task Scheduler for "XMRig Auto Start" task
- Run `start-mining.bat` manually to see errors
- Check `C:\XMRig\xmrig.log` for error messages

**Low hashrate?**
- Enable huge pages: `setup\configure-hugepages.ps1`
- Restart computer after enabling huge pages
- Check CPU temperature (throttling if >85°C)
- Increase `max-threads-hint` in config.json

**Windows Defender blocking?**
- Run `setup\configure-defender.ps1` as Administrator
- Or add manual exclusions for `C:\XMRig`

**No shares accepted?**
- Check internet connection
- Verify wallet address in config.json
- Try alternative pool (supportxmr.com:3333)

For more solutions, see [`docs/TROUBLESHOOTING.md`](docs/TROUBLESHOOTING.md)

## 🛡️ Security Considerations

### Is XMRig Safe?
Yes! XMRig is **legitimate open-source software**. However:
- ⚠️ Often flagged by antivirus (false positive)
- ⚠️ Used by both legitimate miners and malware
- ✅ Only download from official source: https://xmrig.com

### This Project's Safety
- ✅ Downloads XMRig from official GitHub releases
- ✅ All scripts are open source and auditable
- ✅ No network connections except to mining pool
- ✅ No keyloggers, spyware, or malicious code
- ✅ Your wallet is only in local config file

### Antivirus Exclusions
Required to prevent XMRig deletion:
- Added automatically during setup
- Can be added manually if needed
- Only exclude trusted XMRig installation

## 🗑️ Uninstallation

### Clean Uninstall
```powershell
cd C:\XMRig-Automation\tools
.\uninstall.ps1
```

This will:
- Stop mining
- Remove scheduled task
- Remove Windows Defender exclusions
- Optionally delete XMRig directory
- Remove desktop shortcuts

### Manual Uninstall
1. Stop mining: `stop-mining.bat`
2. Delete scheduled task: Task Scheduler → "XMRig Auto Start" → Delete
3. Remove Defender exclusions: Windows Security → Exclusions
4. Delete folders: `C:\XMRig` and `C:\XMRig-Automation`
